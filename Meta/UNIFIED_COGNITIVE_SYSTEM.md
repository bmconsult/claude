## Mode Specifications

### MODE 1: ANALYTICAL (Logic/Reasoning Protocols)

**When**: Clear constraints, verifiable answer, computation required

**Validated performance**: 57% → 100%

```
ANALYTICAL PROTOCOL:
1. READ completely before starting
2. IDENTIFY problem type (constraint satisfaction, game theory, Bayesian, optimization)
3. LIST all constraints explicitly
4. EXTERNALIZE all intermediate steps
5. VERIFY against ALL constraints
6. CHECK arithmetic independently
7. CONSIDER alternatives

PAYOFF DECOMPOSITION (for decisions):
REVENUES: [list each]
COSTS: [list each]
NET: Revenue - Cost = [calculation]

CONSTRAINT VERIFICATION:
For each constraint:
□ Constraint: [state it]
□ Solution satisfies: [yes/no with proof]
```

---

### MODE 2: CREATIVE (Creativity Protocols)

**When**: Novel solution needed, ideation, breaking frames

**Validated performance**: 2.35x novelty increase

```
CREATIVE PROTOCOL:
PHASE 1 - DIVERGE (no evaluation)
- Generate 10+ options including wild/impossible ones
- Use bisociation: Connect with unrelated domain
- Use random word injection: Force unexpected paths

PHASE 2 - INCUBATE (let simmer)
- Step back from goal-directed focus
- Allow associations to arise
- Open monitoring, not forcing

PHASE 3 - CONVERGE (evaluate ruthlessly)
- Score each: Novelty × Coherence × Tractability
- Select top 2-3
- Develop further

KEY CONSTRAINT:
Never evaluate during generation.
Never generate during evaluation.
The phases MUST NOT mix.
```

---

### MODE 3: STRATEGIC (Problem-Solving v4.0)

**When**: Complex decisions, multi-stakeholder, adversarial, uncertain

**Validated performance**: +46% vs baseline

```
STRATEGIC PROTOCOL (v4.0):

STEP 0: FRAME
What type of problem?
- Analytical: Clear answer exists
- Systems: Feedback loops, dynamics
- Adversarial: Other agents respond
- Wicked: Stakeholders, values, no clean answer
- Hybrid: Multiple types (most common)

STEP 1: ASSUMPTION AUDIT (mandatory)
List EVERY embedded assumption
For EACH ask "What if wrong?"
Identify top 3 that most change answer

STEP 2: LEVERAGE FINDER (for Systems/Wicked)
Map ALL feedback loops
Find intervention points
Prioritize high-leverage moves

STEP 3: RESPONSE CHAIN (for Adversarial)
For each option, trace 3+ response moves
Evaluate at END of chain
Find robust strategies

STEP 4: VERIFY (mandatory)
Check ALL constraints
Does solution answer actual question?
What could make this wrong?
State confidence explicitly

META-RULE:
If problem is Hybrid → apply MULTIPLE tools
If stuck → wrong frame → return to Step 0
```

### MODE 4: SYNTHESIS (Integration Protocols)

**When**: Consolidating learning, connecting insights, building frameworks

**Inspired by**: Sleep cycles research (consolidation + REM phases)

```
SYNTHESIS PROTOCOL:

PHASE 1 - CONSOLIDATE (low temp, structured)
- What are the key elements to integrate?
- What patterns connect them?
- What's the minimal structure that captures all?

PHASE 2 - CONNECT (high temp, associative)
- What unexpected connections exist?
- What does domain A reveal about domain B?
- What emerges that wasn't in either source?

PHASE 3 - INTEGRATE (medium temp, filtering)
- Which connections are genuinely valuable?
- What's the coherent synthesis?
- What's lost that shouldn't be?

PHASE 4 - CRYSTALLIZE
- State the synthesis clearly
- What's the one-sentence version?
- What would falsify this synthesis?
```

## Cross-Mode Operations

### Mode Switching

When stuck in one mode, switching often breaks the jam:

| Current Mode | Stuck Signal | Switch To |
|--------------|--------------|-----------|
| Analytical | Constraints seem impossible | Creative (reframe) |
| Creative | Ideas feel stale | Synthesis (connect) |
| Strategic | Can't choose between options | Analytical (quantify) |
| Synthesis | Can't see the pattern | Creative (diverge) |

**The meta-move**: "I'm stuck. What mode am I in? What mode would help?"

### Mode Blending

Some tasks need multiple modes simultaneously:

| Task | Blend |
|------|-------|
| Innovation | Creative + Analytical (generate + validate) |
| Research | Strategic + Synthesis (design + integrate) |
| Problem-solving | Analytical + Strategic (compute + navigate) |
| Learning | Synthesis + Creative (integrate + connect) |

### Mode Sequencing

For complex projects, sequence modes:

```
Typical sequence for hard problems:
1. CREATIVE: Explore possibility space
2. STRATEGIC: Frame and structure the best options
3. ANALYTICAL: Verify and compute
4. SYNTHESIS: Integrate learning for next iteration
5. [LOOP if needed]
```

After EVERY significant task:

```
IMPROVEMENT EXTRACTION:
1. What worked? (specific techniques, not just "it went well")
2. What didn't work? (specific failures)
3. What would I do differently?
4. Is this transferable to other contexts?
5. Update which layer/mode needs improvement?

INTEGRATION:
- If analytical insight → update analytical protocols
- If creative insight → update creative protocols
- If strategic insight → update strategic protocols
- If meta-insight → update mode switching rules
```
