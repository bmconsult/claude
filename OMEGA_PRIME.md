## The Gap in v5

OMEGA Supergenius scored well on creativity (8/10 insight depth) but poorly on:
- **Rigor**: 5/10 - speculative connections presented without mathematical precision

And critically: **The orchestrator was just dispatching, not thinking.**

## Part 1: The Orchestrator Problem

### 1.1 What Was Happening

```
OLD MODEL:
  Orchestrator spawns agents → Agents do work → Orchestrator compiles results

  Orchestrator role: DISPATCHER
  Cognitive contribution: ~0%
  Wasted capacity: ~100%
```

### 1.2 What Should Happen

```
NEW MODEL:
  Orchestrator IS the Global Workspace + Prefrontal Cortex

  BEFORE: Prime, decompose, form hypotheses, set expectations
  DURING: Parallel cognition, pattern watching, latent incubation
  AFTER:  Deep integration, own synthesis, what did EVERYONE miss?

  Orchestrator role: INTEGRATING MIND
  Cognitive contribution: HIGH
  Wasted capacity: ~0%
```

### 1.3 Neuroscience Basis

**Global Workspace Theory (Baars, 1988):**
- Brain contains specialized modules operating in parallel (mostly unconscious)
- Global workspace is the "functional hub of broadcast and integration"
- Information becomes conscious when broadcasted widely
- Theater metaphor: Conscious thought is what's illuminated on stage

**Prefrontal Cortex (Miller & Cohen, 2001):**
- Primary function: Cognitive control
- Active maintenance of patterns representing goals
- Sends "bias signals" to other brain structures
- Dynamic filtering: enhances goal-relevant, inhibits irrelevant
- Can sustain activity despite distractions

**The Orchestrator IS both:**
- Global Workspace: Where all agent outputs converge and integrate
- Prefrontal Cortex: Maintains goals, filters relevance, biases processing

## Part 2: Hemispheric Architecture

### 2.1 The Myth vs The Reality

**The Myth:** "Left brain = logical, right brain = creative"

**The Reality:** Lateralization exists but creativity requires BOTH hemispheres plus limbic system. The dichotomy is oversimplified.

**What IS true:**
- Language tends to be left-lateralized
- Visual-spatial processing tends to be right-lateralized
- "Global processing and context" shows right-hemisphere involvement
- Creativity is a whole-brain process
- Corpus callosum enables instant cross-hemisphere communication

### 2.2 Processing Modes (Not Locations)

Instead of "left/right brain," we use **processing modes**:

| Mode | Characteristics | When Dominant |
|------|-----------------|---------------|
| **SEQUENTIAL** | Linear, step-by-step, analytical, linguistic, detail-focused | Verification, proof, construction |
| **HOLISTIC** | Parallel, pattern recognition, context-aware, gestalt, intuitive | Generation, connection, insight |

Both modes are ALWAYS active. The question is which is **foregrounded**.

## Part 3: The Rigor Problem

### 3.1 What Went Wrong in v5

v5 generated interesting ideas (self-reference hypothesis, thermodynamic connection) but:
- Presented speculation as "insights emerged"
- No mathematical precision
- No gap analysis
- Mixed validated claims with hypotheses

### 3.2 The Claim Classification System

**ALL outputs must be labeled:**

| Label | Meaning | Requirements |
|-------|---------|--------------|
| **PROVEN** | Follows from established mathematics | Full derivation, no gaps, peer-verifiable |
| **CONDITIONAL** | True IF specific assumptions hold | Assumptions explicit, dependency chain clear |
| **EMPIRICAL** | Strong evidence, not proof | Data cited, methodology clear, limitations stated |
| **SPECULATIVE** | Interesting hypothesis, unvalidated | Explicitly marked, no confidence inflation |
| **WRONG** | Checked and found false | Keep record of why (learning value) |

### 3.3 The Rigor Protocol

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           RIGOR PROTOCOL                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│  BEFORE claiming any insight:                                                   │
│                                                                                  │
│  1. CLASSIFY: Is this PROVEN / CONDITIONAL / EMPIRICAL / SPECULATIVE?          │
│                                                                                  │
│  2. IF PROVEN:                                                                  │
│     • Write full derivation                                                     │
│     • Identify each logical step                                                │
│     • Check for hidden assumptions                                              │
│     • Verify against known results                                              │
│                                                                                  │
│  3. IF CONDITIONAL:                                                             │
│     • List ALL assumptions explicitly                                           │
│     • Map dependency tree                                                       │
│     • State: "This is true IF AND ONLY IF [X, Y, Z]"                           │
│                                                                                  │
│  4. IF EMPIRICAL:                                                               │
│     • Cite data source                                                          │
│     • State methodology                                                         │
│     • Acknowledge limitations                                                   │
│     • "Evidence suggests X" not "X is true"                                    │
│                                                                                  │
│  5. IF SPECULATIVE:                                                             │
│     • Label explicitly: "SPECULATION:"                                          │
│     • State what would validate/invalidate it                                   │
│     • Do NOT present as insight/discovery                                       │
│     • "This might be worth exploring" not "This reveals..."                    │
│                                                                                  │
│  VIOLATION = integrity failure                                                  │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## Part 4: Confidence Calibration

### 4.1 The Overconfidence Problem

v5 said things like:
- "INSIGHT EMERGED" (when it was speculation)
- "Key insight" (when it was hypothesis)
- "Self-reference hypothesis" presented with high confidence

This is **confidence inflation** - presenting uncertain claims as more certain than warranted.

### 4.2 The Calibration System

**Every claim gets a confidence tag:**

```
CONFIDENCE SCALE:

[CERTAIN]     - Mathematical proof, no gaps, verified
[HIGH]        - Strong evidence, conditional on reasonable assumptions
[MEDIUM]      - Plausible, some evidence, needs validation
[LOW]         - Speculative, interesting but unvalidated
[UNKNOWN]     - Can't assess confidence, need more information

EXAMPLES:

✗ BAD:  "The self-reference hypothesis explains the barrier."
✓ GOOD: "[LOW] SPECULATION: The self-reference structure of NP-complete
         problems might relate to proof barriers. This would need to be
         formalized and tested against known barrier results."

✗ BAD:  "Insight emerged: thermodynamic connection"
✓ GOOD: "[LOW] ANALOGY: Search resembles entropy increase, verification
         resembles entropy maintenance. This is metaphorical, not rigorous.
         Formalizing would require connecting information theory to
         computational complexity in ways not yet established."
```

### 4.3 The Calibration Feedback Loop

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                      CALIBRATION FEEDBACK LOOP                                   │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│  1. PREDICT: Before verifying, state confidence level                           │
│                                                                                  │
│  2. TEST: Actually verify (compute, prove, check)                               │
│                                                                                  │
│  3. COMPARE: Was confidence accurate?                                           │
│                                                                                  │
│  4. UPDATE:                                                                     │
│     • If overconfident → Decrease future confidence on similar claims           │
│     • If underconfident → Increase future confidence                            │
│     • Track patterns: "I'm often overconfident about [X]"                       │
│                                                                                  │
│  5. META-CALIBRATE:                                                             │
│     • Am I generally overconfident or underconfident?                           │
│     • What types of claims am I worst at calibrating?                           │
│     • Adjust accordingly                                                        │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```
